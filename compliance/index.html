COMPLIANCE FRAMEWORK

四大合規領域

SEOBAIKE 全面遵循台灣法規與國際標準，以下為我們持續執行的四大合規核心。

🔒

個人資料保護

已合規
Personal Data Protection
適用法規
台灣《個人資料保護法》（PDPA）— 規範個人資料之蒐集、處理及利用，保護人民隱私權。本平台作為資料處理者，依法履行告知、同意取得、安全維護等義務。
SEOBAIKE 合規措施
  • 使用者註冊時明確告知資料蒐集目的與範圍
  • 取得使用者書面或電子同意後始得處理個資
  • 實施資料最小化原則，僅蒐集必要資料
  • 建立個資外洩通報機制，於 72 小時內通知主管機關
  • 跨境傳輸符合 PDPA 第 21 條規定
  • 定期進行個資影響評估（PIA）
🤖

AI 治理

已合規
AI Governance
適用法規
台灣《人工智慧基本法》— 規範 AI 系統之開發與應用應遵循透明、可解釋、公平、安全等基本原則，促進負責任的 AI 發展與使用。
SEOBAIKE 合規措施
  • AI 推理路徑受專利 115100981 L1-L4 約束層管控
  • 所有 AI 決策均可追溯，確保透明度與可解釋性
  • 人類決策為主（CaaS 架構），AI 僅為輔助執行者
  • 建立 AI 偏見檢測與公平性驗證機制
  • 定期審計 AI 模型輸出，確保無歧視性結果
  • AI 生成內容均標示為機器生成
⚖️

消費者保護

已合規
Consumer Protection
適用法規
台灣《消費者保護法》及《公平交易法》— 保障消費者知情權、選擇權及公平交易權。網路平台業者應確保資訊透明、提供合理退換機制、禁止不實廣告。
SEOBAIKE 合規措施
  • 服務定價完全透明，無隱藏費用
  • 提供 7 天猶豫期退費機制（依消保法第 19 條）
  • 市集商品上架前經審核，杜絕不實或誇大描述
  • 建立消費爭議調解機制與申訴管道
  • 服務條款以清楚易懂之中文撰寫
  • 禁止不公平競爭行為與綁定銷售
🔐

資訊安全

持續強化中
Information Security
適用法規
台灣《資通安全管理法》— 規範關鍵基礎設施及資通系統之安全管理，要求建立資安防護體系、風險評估機制及事件通報程序。
SEOBAIKE 合規措施
  • 全站 TLS/SSL 加密傳輸，杜絕中間人攻擊
  • 資料庫實施 Row Level Security（RLS）列級安全策略
  • 使用者密碼經 bcrypt 雜湊處理，不儲存明文
  • 建立資安事件應變計畫與通報流程
  • 定期進行弱點掃描與安全稽核
  • 人員存取採最小權限原則，受保密協議約束

SECURITY MEASURES

技術安全措施

SEOBAIKE 採用業界最佳實踐，保護平台及使用者資料之安全性。

🔐

TLS/SSL 全站加密

所有網路通訊均透過 TLS 加密傳輸，確保資料在傳輸過程中的完整性與機密性。

🗄️

Row Level Security

資料庫層級之列級安全策略（RLS），確保使用者僅能存取自己的資料，防止越權存取。

🔑

bcrypt 密碼雜湊

使用者密碼經 bcrypt 雜湊處理後儲存，本公司無法還原明文密碼，即使資料外洩也無法被逆向破解。

📋

定期安全稽核

定期進行安全稽核與弱點掃描，並持續監控異常行為，強化平台之安全防護能力。

🚨

資料外洩應變計畫

備有完整之資料外洩事件應變流程（Incident Response Plan），確保於法定時限內通知受影響使用者及主管機關。

🌐

CDN 安全防護

透過全球 CDN 網路提供 DDoS 防護及效能優化。個人資料不於邊緣節點快取，確保資料隱私。

📜

隱私權政策透明公開

完整之隱私權政策、服務條款公開於網站，使用者可隨時查閱。重大變更時主動通知。

👮

人員存取管控

僅經授權之人員得於業務必要時存取個人資料，採最小權限原則，並受保密協議約束。
