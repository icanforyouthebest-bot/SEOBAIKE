## 1. 版本亮點

### 1.1 核心定位升級

SEOBAIKE v2.0 正式確立「AI 界的 App Store」定位。本版本不再只是一個 AI 工具，而是一個讓各種 AI 服務得以運行的完整平台。核心理念：

- **不做 MCP，做讓 MCP 跑的平台**
- **插拔式架構**：所有 AI 服務可自由上架和下架
- **過路費 + 分潤**：明確的商業模式
- **CaaS 架構**：人類決策為主，AI 為輔助執行

### 1.2 專利約束系統完善

台灣專利 TW-115100981 的 L1-L4 約束層在本版本中達到完整可用狀態：

| 約束層 | 狀態 | 說明 |
|--------|------|------|
| L1 宏觀產業類別 | 完成 | 含多國標準對齊（TSIC/NAICS/NACE/JSIC） |
| L2 次產業分類 | 完成 | FK 關聯至 L1 |
| L3 製程/作業類型 | 完成 | FK 關聯至 L2 |
| L4 原子級工業節點 | 完成 | FK 關聯至 L3 |
| 版本快照 | 完成 | frozen_snapshots 表、RLS 保護 |
| 凍結機制 | 完成 | is_frozen + frozen_at + trigger 保護 |

### 1.3 生態系統擴展

| 項目 | v1.x | v2.0 |
|------|------|------|
| 平台連接器 | 5 | 15 |
| 頻道支援 | 3 | 10 |

## 6. 升級指南

### 6.1 從 v1.x 升級至 v2.0

**步驟一：資料庫更新**
- 執行所有新增的 migration（從 v1.x 最後版本到 #048）
- 驗證 L1-L4 約束表結構
- 確認 RLS 策略更新

**步驟二：Workers 更新**
- 使用 Wrangler 部署最新的 Workers 程式碼
- 更新中介軟體配置
- 設定新增的環境變數

**步驟三：前端更新**
- 在 Framer 中發布最新版本
- 驗證 AI Widget 連線
- 確認生態系統導航功能

**步驟四：驗證**
- 執行端對端測試
- 確認 L1-L4 路徑驗證正常
- 檢查所有 Edge Function 回應

### 6.2 回滾程序

如升級失敗：
1. Workers：Wrangler 回滾至前一版本
2. 資料庫：執行反向 migration
3. 前端：Framer 回復至前一版本
4. 通知 CEO 回滾狀況並提出修復方案

### 6.3 重大變更（Breaking Changes）

| 變更 | 影響 | 遷移方式 |
|------|------|---------|
| API 版本從 v1 升至 v2 | 舊端點將在 3 個月後棄用 | 更新 base URL |
| JWT 格式更新 | 舊 token 需重新簽發 | 用戶重新登入 |
| 約束層 schema 變更 | 部分欄位名稱調整 | 依 migration 自動處理 |
