第一章：系統是什麼？（1:30 - 7:00）
========================================================================

【什麼是 SEOBAIKE？】

你可以把 SEOBAIKE 想像成「AI 界的 App Store」。

就像你手機上的 App Store 不是自己做每一個 App，
而是提供一個平台讓各種 App 可以上架、被下載、被使用。

SEOBAIKE 也是一樣的概念：
我們不是做一個 AI 工具，而是建一個平台，
讓各種 AI 服務都可以在上面跑。

【平台的核心組成】

1. 指揮中心 — 就像大腦，負責接收指令、分配任務
2. 資料庫 — 就像記憶體，記住所有資料和規則
3. 前端介面 — 就像臉，使用者看到的畫面
4. 邊緣運算 — 就像神經，快速傳遞和處理資訊
5. 功能市集 — 就像各種器官，提供不同的功能

【專利約束系統 — L1 到 L4】

這是我們最特別的地方，受到台灣專利 TW-115100981 保護。

想像一個四層的過濾器：
- L1（最上層）：這個任務屬於哪個大產業？例如：科技業、製造業
- L2（第二層）：在這個產業中，屬於哪個細分領域？
- L3（第三層）：具體要做什麼類型的作業？
- L4（最底層）：最精確的一個操作動作

每次 AI 要做事情之前，都必須先通過這四層的檢查。
如果任何一層不通過，AI 就不能執行。
這就是我們說的「專利鎖」。

用生活的例子來說：
假設你要寄一封信——
L1 = 你在哪個國家？（台灣）
L2 = 哪個縣市？（台北市）
L3 = 哪個區？（大安區）
L4 = 哪條路幾號？（信義路四段100號）

只有當四層地址都正確，信才會被寄出去。

【CaaS 架構 — 人指揮，AI 做事】

CaaS 是「Command-as-a-Service」的縮寫，
中文就是「指令即服務」。

核心概念很簡單：
- 人（老闆）下指令
- AI 提出方案
- 人確認「同意」
- AI 才開始做
- AI 做完回報結果

AI 絕對不會自己決定要做什麼。
這是我們公司最重要的規則。

第二章：如何啟動任務？（7:00 - 15:00）
========================================================================

【進入系統】

步驟一：打開瀏覽器
- 在網址列輸入：aiforseo.vip
- 你會看到 SEOBAIKE 的首頁

步驟二：登入
- 點擊右上角的「登入」按鈕
- 使用公司提供的帳號密碼登入
- 登入後會進到主控台（Dashboard）

步驟三：認識主控台
- 左邊是選單列：任務、監控、設定、幫助
- 中間是任務總覽：顯示目前所有任務的狀態
- 右邊是快速操作區：常用功能的捷徑

【建立新任務】

步驟一：點擊「新增任務」按鈕
- 位置在主控台右上方，藍色按鈕
- 點擊後會出現任務建立表單

步驟二：填寫任務內容
- 任務名稱：用簡短的中文描述（例如：分析競爭對手網站）
- 任務說明：詳細描述你想要做什麼
- 優先等級：
  - P0 = 緊急（馬上要）
  - P1 = 高優先（今天內）
  - P2 = 一般（本週內）
  - P3 = 低優先（有空再做）

步驟三：選擇任務類型
- 系統會根據你的描述，自動建議 L1-L4 的分類
- 你只需要確認建議是否正確
- 如果不確定，選「自動分類」讓系統決定

步驟四：提交任務
- 點擊「提交」按鈕
- 系統會顯示：「任務已提交，等待推理路徑驗證」
- 如果 L1-L4 路徑驗證通過，任務會進入佇列
- 如果驗證失敗，系統會告訴你原因

步驟五：等待執行
- 任務提交後，如果是重大變更類任務，
  系統會通知老闆進行核准
- 老闆核准後，AI 才會開始執行
- 一般查詢類任務可以直接執行

【實際操作練習 — 建立你的第一個任務】

現在請跟著我一起做：

1. 點「新增任務」
2. 任務名稱打：「查詢本月流量報告」
3. 說明打：「請幫我查詢本月的網站流量數據，整理成報告」
4. 優先等級選：P2（一般）
5. 點「提交」

很好！你應該會看到一個綠色的通知：
「任務 #XXX 已成功建立」

這就是你的第一個任務！

第三章：如何查看結果？（15:00 - 22:00）
========================================================================

【查看任務進度】

步驟一：進入任務列表
- 點左邊選單的「任務」
- 會看到所有任務的列表

步驟二：認識任務狀態
每個任務旁邊會有狀態標示：
- 等待中（灰色）— 還在排隊
- 驗證中（黃色）— 正在檢查 L1-L4 路徑
- 執行中（藍色）— AI 正在處理
- 已完成（綠色）— 做好了！
- 失敗（紅色）— 出了問題

步驟三：查看任務詳情
- 點擊任何一個任務，會展開詳細資訊
- 你可以看到：
  - 任務的推理路徑（L1 → L2 → L3 → L4）
  - 開始時間和結束時間
  - 消耗的資源量
  - 執行結果

【查看執行結果】

結果會根據任務類型以不同方式呈現：

1. 文字報告類
   - 直接在頁面上顯示報告內容
   - 可以點「下載」保存為 PDF 或 Word

2. 數據分析類
   - 顯示圖表和統計數據
   - 可以切換不同的圖表類型（長條圖、折線圖、圓餅圖）

3. 操作執行類
   - 顯示執行步驟的詳細記錄
   - 每一步是否成功

【查看監控儀表板】

步驟一：點左邊選單的「監控」
步驟二：你會看到即時儀表板，包含：
- 今日任務總數
- 完成率
- 平均回應時間
- 近期錯誤統計

這個儀表板會自動更新，你不需要手動重新整理。

【匯出報告】

如果老闆需要看報告：
1. 在任務列表頁面，選擇要匯出的任務
2. 點右上角的「匯出」按鈕
3. 選擇格式（PDF / Excel / CSV）
4. 檔案會自動下載到你的電腦

第四章：常見問題與解決方法（22:00 - 28:00）
========================================================================

【問題一：任務一直顯示「等待中」】

原因：可能是任務需要老闆核准，或是佇列中排在前面的任務太多。

解決方法：
1. 先檢查任務是否標記了「需核准」
2. 如果是，請通知老闆進行核准
3. 如果不是，等待幾分鐘通常就會開始
4. 如果等了超過 10 分鐘，請聯繫技術團隊

【問題二：任務狀態顯示「失敗」（紅色）】

原因：可能是推理路徑驗證失敗、API 錯誤、或外部服務異常。

解決方法：
1. 點開任務詳情，查看錯誤訊息
2. 常見錯誤訊息：
   - 「INVALID_PATH」→ L1-L4 路徑選擇有誤，請重新建立任務
   - 「PATH_DENIED」→ 這個操作被專利約束禁止了
   - 「PATH_FROZEN」→ 這個類別已被凍結，無法操作
   - 「RATE_LIMITED」→ 請求太頻繁，等一下再試
   - 「INTERNAL_ERROR」→ 系統內部錯誤，聯繫技術團隊
3. 如果不確定怎麼處理，截圖錯誤訊息，回報給你的主管

【問題三：頁面載入很慢或空白】

原因：網路問題或瀏覽器快取。

解決方法：
1. 先確認你的網路連線正常
2. 試著重新整理頁面（按 F5 或 Ctrl+R）
3. 清除瀏覽器快取：
   - Chrome：Ctrl+Shift+Delete → 選擇「快取圖片和檔案」→ 清除
4. 如果還是不行，換一個瀏覽器試試
5. 以上都不行的話，聯繫技術團隊

【問題四：不知道任務該怎麼分類】

原因：剛開始用系統，對 L1-L4 的分類不熟悉。

解決方法：
1. 不用擔心！系統有「自動分類」功能
2. 你只需要用白話文寫出你要做什麼
3. 系統會自動建議分類
4. 如果建議的分類看起來不對，隨時可以問你的主管

【問題五：忘記密碼怎麼辦？】

解決方法：
1. 在登入頁面點「忘記密碼」
2. 輸入你的公司信箱
3. 去信箱收重設密碼的信
4. 點擊信中的連結，設定新密碼
5. 如果收不到信，聯繫 IT 部門

【問題六：看到英文的錯誤訊息看不懂】

解決方法：
1. 截圖！把錯誤訊息截圖下來
2. 去系統的「幫助」頁面，有常見錯誤的中文對照表
3. 或者直接把截圖傳給技術團隊
4. 不要試著自己亂操作「修復」，可能會讓問題更嚴重

【問題七：任務做到一半想取消】

解決方法：
1. 只有「等待中」狀態的任務可以取消
2. 在任務詳情頁面，點「取消任務」按鈕
3. 「執行中」的任務無法中途取消
4. 如果緊急需要停止「執行中」的任務，通知你的主管
