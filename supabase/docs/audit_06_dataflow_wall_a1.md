# MCP 主權 OS

## 資料流（Data Flow）模型

資料是主權。
推論是輔助。
流程是執行。
治理是策略。

---

## 資料流總覽（Top-Level Data Flow）

```
[L1 主權資料層]
        |（唯讀）
        v
[L2 推論與規則層]
        |（推論結果）
        v
[L3 流程與執行層]
        |（事件輸出）
        v
[外部系統 / 金流 / 通知]
```

---

## L1 -> L2（唯讀）

**L1 提供：**

- 使用者
- 商家
- 訂單
- 金流
- 法遵
- 風險
- AML / KYC
- 抽佣 / 反傭

**L2 只能：**

- 讀取
- 套用規則
- 產生 deterministic 推論

**禁止：**

- 寫入 L1
- 控制流程
- 跳層

---

## L2 -> L3（推論結果）

**L2 提供：**

- AML 警示候選
- KYC 驗證結果
- 反傭異常比對
- 風險分數

**L3 只能：**

- 接收推論
- 執行流程
- 更新狀態

**禁止：**

- 修改 L1 結構
- 跳過 L2
- 直接寫入治理層

---

## L3 -> 外部系統（事件驅動）

**L3 觸發：**

- 金流請款
- 通知系統
- 審計紀錄
- 風險標記
- 抽佣計算
- 反傭偵測

**禁止：**

- 直接操作 L1
- 跳層至 L4
- 修改推論結果

---

## 資料流哲學（Data Flow Philosophy）

資料只往上走一次（L1 -> L2）。
推論只往下走一次（L2 -> L3）。
流程只往外走一次（L3 -> External）。

沒有回頭路。
沒有跨層路。
沒有隱藏路。

---

## 資料流三大原則

**1. 單向（Unidirectional）**

資料永遠往上，推論永遠往下，流程永遠往外。

**2. 不可逆（Irreversible）**

任何層級不得逆向操作下層。

**3. 不可污染（Non-Contamination）**

資料層永不被邏輯污染。
推論層永不被流程污染。
流程層永不被治理污染。

---

## MCP 主權 OS 的資料流信念

資料是根。
推論是枝。
流程是葉。
治理是光。

四層分離，資料流永不混亂。
這是主權 OS 的秩序。
