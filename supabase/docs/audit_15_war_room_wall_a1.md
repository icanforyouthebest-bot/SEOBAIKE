# MCP 主權 OS

## WAR ROOM EDITION

**戰情室版（決策核心牆）**

---

## 一、戰場原則（Battlefield Principles）

邊界先於策略。
策略先於流程。
流程先於邏輯。
邏輯先於工程。

任何違反順序的行為
都是風險、都是污染、都是敗因。

---

## 二、主權鐵律（Sovereignty Iron Laws）

**鐵律 1：資料不可變（L1）**

資料一旦定義，任何人不得修改其語義。

**鐵律 2：推論唯讀（L2）**

推論永遠不能寫入資料，也不能控制流程。

**鐵律 3：流程不可跳（L3）**

流程必須依賴推論，不能越權、不能跳層。

**鐵律 4：治理不可觸（L4）**

治理不能直接操作資料，也不能繞過流程。

---

## 三、戰情室核心圖（War Room Core Map）

```
L4 — Governance
      | Strategy
      v
L3 — Process
      | Execution
      v
L2 — Inference
      | Read-Only
      v
L1 — Data Sovereignty
```

單向。
不可逆。
不可污染。

---

## 四、戰情室禁止路徑（Critical No-Go Zones）

```
L4 -> L1  X
L4 -> L2  X
L3 -> L1（修改結構） X
L2 -> L1（寫入） X
L2 -> L3（流程控制） X
L1 -> 上層引用 X
```

任何越權都是攻擊面。
任何跳層都是漏洞。
任何污染都是崩壞。

---

## 五、戰情室三問（The Three War Questions）

每一個決策、每一個模組、每一個流程
都必須回答這三問：

**1. 它屬於哪一層？（L1-L4）**

如果無法回答 -> 立即停止。

**2. 它是否越權？**

如果越權 -> 立即封鎖。

**3. 它是否污染資料？**

如果污染 -> 立即移除。

---

## 六、戰情室行動準則（Operational Directives）

**Directive 1：所有責任必須可審計**

不可有黑箱、不可有隱藏流程。

**Directive 2：所有模組必須可替換**

供應商不可成為單點風險。

**Directive 3：所有邊界必須可驗證**

任何模糊都是風險。

**Directive 4：所有流程必須可重建**

系統必須能在戰場上重生。

---

## 七、戰情室精神（War Room Spirit）

清晰比速度重要。
邊界比功能重要。
主權比便利重要。

---

## 八、戰情室宣言（War Room Manifesto）

我們不依賴人。
我們不依賴供應商。
我們不依賴工程師。

我們依賴邊界。
我們依賴資料。
我們依賴主權。
