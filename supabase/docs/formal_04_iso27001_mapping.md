## 4. ISO 27001 → MCP 主權 OS 的核心對應原則

### 原則 1：ISO 控制項 = MCP 邊界語義化

所有 ISO 控制項都被 MCP 語義層（L5–L8）轉換成不可越權的邊界。

### 原則 2：ISO 稽核 = MCP 不可變資料 + 全面追溯

所有控制項都有：

- 不可變資料（L1）
- 不可變日誌（L1）
- 流程重建（L3）
- 推論重建（L2）
- 治理決策重建（L4）

### 原則 3：ISO 委外要求 = MCP 供應商不可越權

所有供應商控制項（5.19–5.23）在 MCP 中自動強制：

- 供應商不可觸碰資料
- 供應商不可寫入推論
- 供應商不可跳層
- 供應商模組可替換
