# MCP 主權 OS

## 四層治理模型（L1-L4）

**Sovereign Operating System — Governance Wall**

---

## L1 — 主權資料層

**Sovereign Data Layer**

原則：不可變、不可污染、不可跨層。
所有責任由資料定義。
所有邊界由資料決定。
所有模組以資料為唯一真實來源（Single Source of Truth）。

**職權：**

- 定義資料結構
- 定義法遵資料
- 定義不可變邊界
- 不含邏輯、不含流程、不含推論

**禁止：**

- 不得引用 L2/L3/L4
- 不得包含任何商業邏輯
- 不得包含流程控制
- 不得包含推論或模型

---

## L2 — 推論與規則層

**Inference & Rules Layer**

原則：唯讀、可解釋、不可寫入。
推論只能讀資料。
推論不能改資料。
推論不能控制流程。

**職權：**

- 讀取 L1
- 套用規則
- 產生 deterministic 推論

**禁止：**

- 不得寫入 L1
- 不得觸發流程
- 不得引用 L3/L4

---

## L3 — 流程與執行層

**Process Execution Layer**

原則：事件驅動、狀態變更、永不跳層。
流程只能依賴推論。
流程不能跳過推論。
流程不能修改資料結構。

**職權：**

- 串接 L1 與 L2
- 執行事件驅動流程
- 控制狀態變更

**禁止：**

- 不得修改 L1 結構
- 不得引用 L4
- 不得跳過 L2 推論

---

## L4 — 治理與自動化層

**Governance & Automation Layer**

原則：策略高於流程，治理高於邏輯。
治理不能直接操作資料。
治理不能跳過流程。
治理不能污染推論。

**職權：**

- 策略
- 治理
- 自動化
- 跨模組協作

**禁止：**

- 不得直接操作 L1
- 不得跳過 L3
- 不得寫入推論層

---

## 禁止路徑（Prohibited Paths）

```
L4 -> L1  X
L4 -> L2  X
L3 -> L1（修改結構） X
L2 -> L1（寫入） X
L2 -> L3（流程控制） X
L1 -> 上層引用 X
```

## 允許路徑（Allowed Paths）

```
L1 -> L2（唯讀）
L2 -> L3（推論結果）
L3 -> 外部系統（事件）
L4 -> L3（治理策略）
```

---

## MCP 主權 OS 的核心信念

資料是主權。
推論是輔助。
流程是執行。
治理是策略。

四層分離，邊界不可破。
這是主權 OS 的根本。
