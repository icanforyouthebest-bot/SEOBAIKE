# MCP 主權 OS

## 禁止路徑（Prohibited Paths）

邊界不可破。
層級不可跳。
資料不可污染。

---

## 總原則（Prime Directive）

任何層級不得越權操作下層。
任何層級不得跳過中間層。
任何層級不得污染資料層。

---

## 禁止路徑總覽（Prohibited Path Map）

```
L4 --X--> L1
L4 --X--> L2
L3 --X--> L1（修改結構）
L2 --X--> L1（寫入）
L2 --X--> L3（流程控制）
L1 --X--> 上層引用
```

---

## L1（主權資料層）禁止：

- 不得引用 L2
- 不得引用 L3
- 不得引用 L4
- 不得包含邏輯
- 不得包含流程
- 不得包含推論

L1 是主權，不接受任何上層干預。

---

## L2（推論與規則層）禁止：

- 不得寫入 L1
- 不得觸發流程
- 不得引用 L3
- 不得引用 L4
- 不得修改資料結構

L2 只能讀，不得寫。
L2 只能推論，不得執行。

---

## L3（流程與執行層）禁止：

- 不得修改 L1 結構
- 不得跳過 L2 推論
- 不得引用 L4
- 不得直接操作資料層

L3 是執行，不是治理。
L3 必須依賴推論，不得自作主張。

---

## L4（治理與自動化層）禁止：

- 不得直接操作 L1
- 不得跳過 L3
- 不得寫入 L2
- 不得控制資料層

L4 是策略，不是工程。
治理永遠不能碰資料。

---

## 禁止路徑哲學（Prohibited Path Philosophy）

越權即污染。
跳層即失控。
混層即崩壞。

主權 OS 的穩定性，
來自於每一條禁止路徑的堅守。

---

## 一句話總結

禁止路徑不是限制，是主權。
邊界不是阻擋，是秩序。
