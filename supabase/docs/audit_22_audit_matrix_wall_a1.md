---

## 一、審計宇宙（Audit Universe）

所有審計都來自四件事：

資料（L1）
推論（L2）
流程（L3）
治理（L4）

沒有第五層。
沒有例外。

---

## 二、審計矩陣總覽（Audit Matrix Overview）

|  | L1（資料） | L2（推論） | L3（流程） | L4（治理） |
|--|-----------|-----------|-----------|-----------|
| 可追溯性 Trace | 最高 | 高 | 高 | 中 |
| 可驗證性 Verify | 最高 | 高 | 高 | 中 |
| 可重建性 Rebuild | 最高 | 高 | 高 | 低 |
| 可審計性 Audit | 最高 | 高 | 高 | 中 |

L1 是審計根。
L2 是審計邏輯。
L3 是審計流程。
L4 是審計治理。

---

## 三、審計 1：資料審計（L1 Data Audit）

資料是唯一真實來源（SSOT）。

**必須審計：**

- 欄位定義
- 欄位語義
- 欄位不可變性
- 欄位責任歸屬
- 欄位法遵要求

**不得發生：**

- 邏輯覆寫
- 流程污染
- 治理干預

資料污染 = 審計失效。

---

## 四、審計 2：推論審計（L2 Inference Audit）

推論是規則，不是權力。

**必須審計：**

- 規則來源
- 規則版本
- 規則語義
- 推論輸入
- 推論輸出

**不得發生：**

- 寫入資料
- 控制流程
- 隱藏邏輯

推論越權 = 治理崩壞。

---

## 五、審計 3：流程審計（L3 Process Audit）

流程是執行，不是治理。

**必須審計：**

- 事件來源
- 狀態變更
- 流程路徑
- 執行責任
- 外部 API

**不得發生：**

- 跳過推論
- 修改資料結構
- 黑箱流程

流程跳層 = 系統失控。

---

## 六、審計 4：治理審計（L4 Governance Audit）

治理是策略，不是工程。

**必須審計：**

- 治理決策
- 治理依據
- 治理版本
- 治理邊界
- 治理授權

**不得發生：**

- 直接操作資料
- 直接寫入推論
- 跳過流程

治理越權 = 主權破壞。

---

## 七、審計三問（Audit Questions）

每一個模組、每一條流程、每一段邏輯
都必須回答：

**Q1：它能被追溯嗎？**

不能 -> 風險。

**Q2：它能被驗證嗎？**

不能 -> 黑箱。

**Q3：它能被重建嗎？**

不能 -> 事故。
