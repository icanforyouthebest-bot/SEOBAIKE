## 一、決策矩陣（Decision Matrix）

所有決策必須回答三個問題：

### Q1：它屬於哪一層？（L1-L4）

```
L1 -> 資料
L2 -> 推論
L3 -> 流程
L4 -> 治理
```

如果無法分類 -> 立即停止。

### Q2：它是否越權？

```
L2 -> L1（寫入） X
L3 -> L1（修改結構） X
L4 -> L1（直接操作） X
L3 -> L2（跳過推論） X
L4 -> L3（跳過流程） X
```

越權 = 污染。
污染 = 崩壞。

### Q3：它是否污染資料？

資料污染 = 系統死亡

任何污染 -> 立即移除。

## 二、邊界矩陣（Boundary Matrix）

每一層的邊界必須清晰、可審計、不可破壞。

### L1 — 資料邊界（Data Boundary）

**允許：**
- 定義資料
- 定義法遵
- 定義不可變欄位

**禁止：**
- 邏輯
- 流程
- 推論
- 治理

### L2 — 推論邊界（Inference Boundary）

**允許：**
- 讀取資料
- 套用規則
- 產生推論

**禁止：**
- 寫入資料
- 控制流程
- 修改結構

### L3 — 流程邊界（Process Boundary）

**允許：**
- 事件驅動
- 狀態變更
- 執行流程

**禁止：**
- 跳過推論
- 修改資料結構
- 直接治理

### L4 — 治理邊界（Governance Boundary）

**允許：**
- 策略
- 治理
- 自動化

**禁止：**
- 觸碰資料
- 跳過流程
- 寫入推論

---

## 三、風險矩陣（Risk Matrix）

所有風險都來自三件事：

### 風險 1：越權（Overreach）

```
L2 -> L1
L3 -> L1
L4 -> L1
L4 -> L2
L4 -> L3
```

越權 = 攻擊面。
攻擊面 = 漏洞。

### 風險 2：跳層（Layer Skipping）

```
流程跳過推論
治理跳過流程
推論跳過資料
```

跳層 = 失控。
失控 = 系統崩壞。

### 風險 3：污染（Contamination）

```
資料被邏輯污染
推論被流程污染
流程被治理污染
```

污染 = 死亡。
這是 OS 最大的敵人。
